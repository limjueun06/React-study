✔ React Project 생성하는 방법
1. 리액트 프로젝트를 생성하고자 하는 폴더로 vscode를 킨다.
2. npm create vite@latest 프로젝트이름 -- --template react
    ※ 주의사항 : -- template 사이를 띄우면 안된다.
3. Use ~ ? -> No
    - 새버전 쓸래? 이런 이야기임
4. 설치하고 바로 실행할래? => Yes

✔ React Project를 실행하는 방법
 - 나의 상황에 따라 다르게 행동

 case1. 서버가 켜져있다. (터미널에 값을 입력 할 수 없는 상태) 
    1-1. 만약 켜져있는 서버가 본인이 희망하는 프로젝트라면 그대로 실행
        => localhost : 포트번호로 이동하면 됨

    1-2. 그게 아니라면? Ctrl+C 연타해서 서버 꺼주기
    1-3. cd + 경로 (내가 키고 싶은 프로젝트)
    1-4. npm run dev

 case2. 내 터미널의 마지막 경로가 react 다 (== 내 노드모듈이 있는 위치가 아니다.)
    1-3, 1-4 진행
    
 case3. 내 터미널의 마지막 경로가 project01이다. (== 내 노드모듈이 있는 위치다)
    1-4 진행

win + . : 😊사용



⚠ ERROR CASE !
- React 는 JS를 보완하는 라이브러리다, 프론트엔드의 영역임
- 반드시 브라우저의 F12 개발자 도구를 통해 오류를 확인
- 오류메시지, 오류가 난 위치를 우선 확인할 것

[1] '~~~' is not defined.
 - 내가 쓰려고 하는 그 기능 / 변수 가 선언되지 않은 경우
 - import 하지 않은 경우
 - 오타가 난 경우

[2] Object is not iterable~
    Object are not valid as a React child ~

  - 어딘가에서 {}, [] 괄호를 잘못 적은 경우
  - props 를 보낸 경우 매개변수 자리에 ({속성1, 속성2}) 이렇게 써야하는데
    (속성1, 속성2) 이렇게 바로 쓰거나
    ([속성1, 속성2]) 이렇게 쓰는 경우 구조적으로 오류

[3] Module not found : Error Can't resolve '~~' in '~~'
    - 모듈을 설치해야하는데 설치하지 않은것
    - "저 설치 했는데요?" => 설치 할때 오타가 났거나, 불러 올때 오타가 난 경우
    - "저 설치 한지 안한지 모르겠는데요?" => 리액트의 package.json 들어가면 확인가능

[4] export 'default' was not found in '경로'
    - 경로 안에서 해당 컴포넌트를 찾을 수가 없음
    - 파일이 없는 경우
    - 파일은 있는데 밑에 export를 적지 않은 경우
    - 실수로 파일을 소문자로 만들었다가 대문자로 이름을 바꾼 경우 리액트 프로젝트가 파일을
      일시적으로 인식하지 못함

[5] 아무 오류도 없이 화면이 사라짐
    - 괄호 실수로 인하여 return 문 위치에 이상이 생긴경우
    - 괄호를 고치다가 다른 특정 함수 안에 컴포넌드가 포함되어버린 경우
    - main.jsx 파일에서 아예 불러오지 않은 경우
